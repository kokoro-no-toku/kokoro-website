Kokoro no Talk API

Welcome to the Kokoro no Talk (こころのトーク) API. This REST API allows you to integrate our Japanese AI mental support service — powered by Google Gemini — into your applications, platforms, and workflows.

日本語・English の両言語に対応しています。All responses are UTF-8 encoded JSON.

Base URL:  https://api.example.com/v1
All API requests must be made over HTTPS.

Quick Start

cURL example 
# Send a chat message to the AI
curl -X POST "https://api.example.com/v1/chat" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "user_abc123",
    "message": "最近、仕事のことで悩んでいます。",
    "language": "ja"
  }'

Authentication

All API requests require a Bearer token in the Authorization header. API keys are generated from your dashboard after subscription.

Keep your API key secret. Never expose it in client-side code or public repositories.
Authorization header 
Authorization: Bearer knt_live_xxxxxxxxxxxxxxxxxxxxxxxx

API Key Format

PrefixEnvironmentDescription
knt_live_ProductionLive API key for production use
knt_test_SandboxTest key — no charges, mock responses

LINE Webhook

POST /webhook/line LINE platform → your server

This endpoint receives events from the LINE Messaging API. Configure this URL in your LINE Developer Console as your webhook URL. The service automatically processes incoming messages and replies via LINE.

LINE will verify your server with a X-Line-Signature header. Signature validation is handled automatically.

Webhook Event Types Handled

Event TypeDescription
messageUser sends a text message — triggers AI response
followUser adds the LINE Official Account
unfollowUser blocks or removes the account
postbackUser taps a quick reply button

/chat

POST /v1/chat Send a message to the AI

Send a message to the Kokoro no Talk AI and receive an empathetic, contextually-aware response in Japanese.

Request Parameters

ParameterTypeDescription
user_id requiredstringLINE user ID or internal user identifier
message requiredstringUser's message text (max 2000 chars)
language optionalstringLanguage code: ja (default) or en
session_id optionalstringConversation session ID for context continuity
emotion_analysis optionalbooleanReturn emotion analysis in response (default: true)

SDKs & Libraries

Official SDKs are under development. Current supported integration methods:

Python (Flask integration example) 
# Install dependencies
pip install flask line-bot-sdk google-generativeai paypalrestsdk psycopg2

# Basic webhook handler
from flask import Flask, request, abort
from linebot import LineBotApi, WebhookHandler
import google.generativeai as genai

app = Flask(__name__)
line_bot_api = LineBotApi('YOUR_CHANNEL_ACCESS_TOKEN')
handler = WebhookHandler('YOUR_CHANNEL_SECRET')
genai.configure(api_key='YOUR_GEMINI_API_KEY')

@app.route('/webhook/line', methods=['POST'])
def webhook():
    signature = request.headers['X-Line-Signature']
    body = request.get_data(as_text=True)
    handler.handle(body, signature)
    return 'OK'
